T06 — Servicii și Integrări Externe

Toate serviciile se află în app/Services/. Fiecare clasă are o singură responsabilitate și nu cunoaște alte servicii (cu excepția ChatService care orchestrează mai multe modele).

Serviciu: ClaudeAiClient

Fișier: app/Services/Ai/ClaudeAiClient.php

Scop: Client centralizat pentru Anthropic API. Oferă apeluri simple (text completion) și acces la clientul brut pentru tool-use/streaming. Calculează automat costul USD pe baza token-ilor consumați.

Dependențe constructor:

  • ?string $apiKey — opțional; dacă lipsește, se citește din AppSetting::KEY_ANTHROPIC_API_KEY (criptat în DB), cu fallback pe env('ANTHROPIC_API_KEY').

Metode publice:

MetodăParametriReturnDescriere
rawClient() — AnthropicClient Returnează instanța SDK Anthropic brută. Folosit pentru tool-use, streaming sau parametri avansați.
complete() string $prompt, ?string $system, string $model, int $maxTokens array{content, tokens, cost_usd} Trimite un mesaj simplu și returnează răspunsul complet cu numărul de tokeni și costul calculat în USD.
calculateCost() (static) string $model, int $inputTokens, int $outputTokens float Calculează costul USD pe baza tabelului de prețuri intern. Prețuri: Sonnet 4.6 — $3/$15 per M tokeni; Haiku — $0.80/$4.00 per M tokeni.
pricing() (static) string $model array{input: float, output: float} Returnează tabelul de prețuri (USD per milion tokeni) pentru modelul dat.
modelSonnet() (static) — string Returnează ID-ul modelului Sonnet din config (default: claude-sonnet-4-6).
modelHaiku() (static) — string Returnează ID-ul modelului Haiku din config (default: claude-haiku-4-5-20251001).

Integrare externă:

API URL: https://api.anthropic.com/v1/messages (via SDK PHP anthropic-sdk-php)
Autentificare: Header x-api-key — cheie stocată criptat în AppSetting::KEY_ANTHROPIC_API_KEY
Modele disponibile: claude-sonnet-4-6 (calitate înaltă), claude-haiku-4-5-20251001 (rapid/ieftin)
Rate limits: Nu sunt implementate explicit — gestionare prin retry manual în serviciile consumatoare.
Constante fallback: MODEL_SONNET = 'claude-sonnet-4-6', MODEL_HAIKU = 'claude-haiku-4-5-20251001'

Retry logic:

Metoda privată retryRequest(callable $fn, int $tries=3) reîncearcă automat la cURL error 28 (timeout) și HTTP 503. Între încercări: usleep(500_000 × attempt) (0.5s, 1s). Orice altă excepție se reraise imediat.

Integrare externă:

API URL: {base_url}/wp-json/wc/v3/
Autentificare: HTTP Basic Auth cu consumer_key / consumer_secret (OAuth WooCommerce)
SSL: verificare configurabilă per conexiune (verify_ssl)
Timeout: configurat per IntegrationConnection::resolveTimeoutSeconds()

Serviciu: ImapMailboxService

Fișier: app/Services/Email/ImapMailboxService.php

Scop: Serviciu IMAP pur — conectare, listare foldere, fetch și parsare emailuri. Nu persistă nimic în DB. Responsabilitatea se oprește la returnarea datelor brute structurate. Folosit de FetchEmailsJob.

Dependențe constructor:

  • string $host, int $port, string $encryption, string $username, string $password, bool $validateCert=true

Factory method fromAppSettings() construiește instanța din AppSetting (cheile KEY_IMAP_*). Returnează null dacă credențialele lipsesc.

Metode publice:

MetodăParametriReturnDescriere
connect() — static Conectează la serverul IMAP via biblioteca webklex/php-imap. Returnează $this pentru method chaining.
disconnect() — void Deconectează și eliberează resursele. Erorile la deconectare sunt ignorate silențios.
reconnect() — static Deconectare + gc_collect_cycles() + reconectare. Util între foldere pentru a elibera starea IMAP internă.
getFolders() — string[] Returnează path-urile folderelor disponibile, excluzând automat trash/spam/junk/drafts (case-insensitive pe suffix).
searchUidsSince() string $folder, DateTimeInterface $since string[] Returnează UID-urile mesajelor din folder care au SINCE data dată. Folosește IMAP::ST_UID cu search direct pe conexiune (nu ->all() care crăpă pe unele servere).
fetchMessage() string $folder, string $uid ?array Fetch și parsare un singur email după UID. Marchează mesajul ca leaveUnread(). Returnează null dacă UID-ul nu există.
fetchMessages() string $folder, string[] $uids array[] Fetch mai multor UID-uri. UID-urile lipsă sunt omise silențios; erorile Duplicate entry sunt ignorate.
fetchSince() string $folder, DateTimeInterface $since, string[] $existingUids=[] array{uids, messages} Combină searchUidsSince() + filtrare UID-uri deja în DB + fetchMessages(). API convenabil pentru FetchEmailsJob.
fromAppSettings() (static) — ?static Factory: construiește instanța din AppSetting. Returnează null dacă host/user/password lipsesc.

Structura mesajului parsat:

{
  uid, folder, from_email, from_name, subject,
  body_html, body_text, sent_at,
  to: [{email, name}],
  cc: [{email, name}],
  attachments: [{name, size, mime_type}]
}

Integrare externă:

Protocol: IMAP via webklex/php-imap
Credențiale: Stocate în AppSetting — KEY_IMAP_HOST, KEY_IMAP_PORT (default 993), KEY_IMAP_ENCRYPTION (default ssl), KEY_IMAP_USERNAME, KEY_IMAP_PASSWORD (criptat)
Foldere excluse automat: trash, spam, junk, drafts
Decodare subiecte: MIME encoded-word (=?UTF-8?Q?...?=) via mb_decode_mimeheader()

Serviciu: ChatService

Fișier: app/Services/ChatService.php

Scop: Chatbot AI pentru site-ul public malinco.ro. Gestionează sesiunile de conversație (cache Redis/file), apelează Claude cu Tool Use în loop și returnează răspunsul text + carduri produs pentru widget.

Dependențe constructor:

Fără parametri. Citește AppSetting::KEY_ANTHROPIC_API_KEY la inițializare.

Configurare (config/app.php — secțiunea malinco):

  • chat.max_history — numărul maxim de mesaje salvate în cache per sesiune (default 20)
  • chat.session_ttl — TTL cache sesiune în secunde (default 900 = 15 min)
  • cache.chat_prefix — prefix cache istoric: chat:
  • cache.chat_state — prefix cache stare conversație: chat_state:

Metode publice:

MetodăParametriReturnDescriere
chat() string $sessionId, string $userMessage, ?array $pageContext, ?array $pageHistory array{reply, products, input_tokens, output_tokens, show_contact_form, contact_form_message} Entry point principal. Algoritm:
  1. Verifică limita cost per sesiune.
  2. Mesaje triviale (mulțumesc, etc.) — răspuns instant fără apel AI.
  3. Apelează callClaude() cu istoricul și contextul paginii.
  4. Salvează istoricul și costul; logează în AiUsageLog.

Tools disponibile (Tool Use Claude):

ToolInputDescriere
searchProducts query, category?, brand? Caută produse publicate în catalog WooCommerce (fulltext + LIKE). Populează $foundProducts pentru widget. Returnează max 5 produse cu id, name, price, image_url, url, stock_available, unit.
getAvailableBrands query, category? Returnează brandurile disponibile pentru un tip de produs, cu numărul de produse per brand.
checkQuantityAvailability product_query, quantity Verifică disponibilitatea unui produs pentru o cantitate dată. Nu returnează stoc exact — răspuns calitativ (disponibil/insuficient).
getOrderStatus email, order_number? Caută comenzi WooCommerce după email client (și opțional număr comandă). Returnează status, total și data comenzii.
getCompanyInfo topic Returnează informații despre companie: program, adresă, contact, transport, plată — citite din AppSetting.
collectContactInfo email?, phone?, wants_specialist Salvează datele de contact în ChatContact și trimite notificare Telegram via SendTelegramLeadJob.
showContactForm message Semnalizează widget-ului să afișeze formularul de contact. Executat o singură dată per sesiune.

Limite și protecții:

  • Max 5 iterații tool-use per request (MAX_TOOL_ITERATIONS)
  • Max 6 mesaje trimise la Claude din istoric (MAX_CONTEXT_MESSAGES)
  • Limită cost per sesiune (configurabilă)
  • Mesaje triviale detectate fără apel AI
  • showContactForm dezactivat după prima apariție în starea conversației

Integrare externă:

API URL: https://api.anthropic.com/v1/messages
Model: claude-haiku-4-5-20251001 (rapid și ieftin pentru chat)
Timeout: 30s per apel HTTP
Max tokens per răspuns: 768

Dependențe constructor:

Fără parametri. Citește JSON-ul contului de serviciu Google de la calea configurată în AppSetting::get('google_service_account_path') (default: /var/www/scripts/cegim/service_account.json).

Metode publice:

MetodăParametriReturnDescriere

Serviciu: StyleAnalysisService

Fișier: app/Services/SocialMedia/StyleAnalysisService.php

Scop: Analizează postările istorice de pe o pagină Facebook (text + imagini reale) și generează un profil de stil (SocialStyleProfile) via Claude Vision. Profilul este folosit ulterior de SocialCaptionService pentru a menține consistența tonului.

Dependențe constructor:

Fără parametri. Citește AppSetting::KEY_ANTHROPIC_API_KEY la apel.

Metode publice:

MetodăParametriReturnDescriere
analyzeAndSave() SocialAccount $account, array $posts SocialStyleProfile Algoritm:
  1. Selectează max 50 postări cu text.
  2. Descarcă max 10 imagini ca base64 pentru Claude Vision.
  3. Trimite request multimodal Claude Sonnet (text + imagini reale).
  4. Parsează răspunsul JSON: tone, vocabulary, hashtag_patterns, caption_structure, visual_style.
  5. Dezactivează profilul anterior activ și salvează noul profil.

Câmpuri profil stil:

  • tone — descriere ton: formal/informal, entuziast/neutru, tip limbaj
  • vocabulary — cuvinte recurente, lungime text, emoji-uri
  • hashtag_patterns — tipuri hashtag-uri, număr mediu
  • caption_structure — structura postărilor: hook, CTA, contact
  • visual_style — descriere detaliată stil vizual real (folosit ca prompt pentru Gemini)

Integrare externă:

API URL: https://api.anthropic.com/v1/messages
Model: Claude Sonnet (cu Vision pentru imagini)
Timeout: 120s (analiza imaginilor poate dura)
Max tokens: 4096
Log usage: AiUsageLog::record('social_style_analysis', ...)

Serviciu: TemplateGeneratorService

Fișier: app/Services/SocialMedia/TemplateGeneratorService.php

Scop: Pipeline în 4 pași pentru generarea automată de GraphicTemplate-uri din imagini de referință. Claude analizează structura vizuală, backend-ul aplică rețete de layout pre-proiectate, apoi rafinare stil și normalizare layout.

Dependențe constructor:

Fără parametri. Citește AppSetting::KEY_ANTHROPIC_API_KEY la apel.

Pipeline (4 pași):

  1. STEP A — Analiză Claude Vision: Detectează familia de compoziție, măsoară zonele reale (procente din canvas), detectează elemente decorative (diagonale, panouri colorate), extrage culori brand și semnale de stil vizual.
  2. STEP B — Rețete de layout: Fiecare familie de compoziție are o rețetă pre-proiectată cu coordonate sigure. Elementele opționale sunt incluse doar dacă Claude le-a detectat.
  3. STEP C — Rafinare stil: Ajustează proporțiile elementelor pe baza semnalelor de stil (mărimea logo, dominanța produsului, prominența footer-ului, scala titlului).
  4. STEP D — Normalizare layout: Constrângeri hard: footer ancorat jos, margini sigure, prevenire suprapuneri. Elementele decorative sunt excluse din constrângeri.

Familii de compoziție (RECIPES):

  • product_right_text_left — produs dreapta, text+logo stânga
  • product_left_text_right — produs stânga, text+logo dreapta
  • centered_product — produs centrat, titlu deasupra
  • partner_campaign — dual brand (Malinco + partener), produs jos
  • text_announcement — fără produs, text dominant
  • promo_badge — produs cu badge promoțional proeminent

Metode publice:

MetodăParametriReturnDescriere
analyzeAndGenerate() string[] $imagePaths — căi relative în disk public GraphicTemplate[] Entry point principal. Acceptă max 8 imagini. Trimite imaginile (base64) + promptul structural la Claude Sonnet, parsează răspunsul JSON, aplică pipeline-ul complet (Steps B-D) și persistă template-urile generate în DB.

Profile de stil default (Malinco baseline):

  • title_size: large, title_weight: extrabold (Montserrat 800+)
  • logo_size: medium, product_dominance: high
  • footer_prominence: strong, spacing_feel: balanced, accent_style: moderate

Integrare externă:

API URL: https://api.anthropic.com/v1/messages
Model: Claude Sonnet (cu Vision)
Timeout: 120s
Max tokens: 4096
Max imagini per apel: 8
Log usage: AiUsageLog::record('template_generation', ...)
