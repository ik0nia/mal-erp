1. purchase_requests

Scop: Necesare de achiziție (PNR). Un utilizator dintr-o locație creează un necesar și îl trimite buyer-ilor pentru a fi acoperit prin comenzi de achiziție.

Indecși & FK:

  • UNIQUE (number)
  • FK user_id → users(id) — restricție standard
  • FK location_id → locations(id) — restricție standard
  • FK woo_order_id → woo_orders(id) — nullOnDelete

Relații Eloquent:

  • user() → BelongsTo User
  • location() → BelongsTo Location
  • items() → HasMany PurchaseRequestItem
  • wooOrder() → BelongsTo WooOrder

Logică business:

  • getOrCreateDraft(User) — firstOrCreate draft per user (previne duplicate)
  • recalculateStatus() — recalculează statusul pe baza itemelor: dacă toți ordered_qty ≥ qty → fully_ordered; dacă cel puțin unul > 0 → partially_ordered; altfel → submitted; dacă toți cancelați → cancelled
  • createFromWooOrder(WooOrder) — auto-generare necesare pentru produse on_demand (idempotent)
