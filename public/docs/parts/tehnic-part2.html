NecesarMarfa (Raport Necesar de Marfă)

Grup: Rapoarte, sort 0. Permisiune: RolePermission::check(). View: filament.app.pages.necesar-marfa.

Algoritmul de calcul

Pagina interogează tabelele product_suppliers, woo_products, product_stocks (agregat per produs) și bi_product_velocity_current (view BI cu medii zilnice calculate). Cache 5 minute per user + coverDays.

Query principal

Condiție de includere: stock < MAX(avg_out_qty_7d, avg_out_qty_30d) × fetchDays (unde fetchDays = max(coverDays, 14)). Se execută separat pentru produse cu furnizor activ și fără furnizor, unite cu UNION.

Calcul dinamic per produs

// Baseline: maximul dintre mediile zilnice 7d, 30d, 90d
$base = max($avg7, $avg30, $avg90);

// Trend: atenuare conservatoare dacă cererea scade >15% față de media 30d
$trend = 1.0;
if ($avg30 > 0 && $avg7 > 0 && $avg7 < $avg30 * 0.85) {
    $trend = max(0.5, $avg7 / $avg30);
}

$adjustedDaily = $base * $trend;
$safetyStock   = $adjustedDaily * 3;  // buffer 3 zile

// Cantitate recomandată de comandat:
$recommended_qty = max(0, ceil($adjustedDaily * $coverDays + $safetyStock - $stock));

// Zile până la epuizare (folosind avg_daily_7d ca estimare curentă):
$days_until_stockout = $avg7 > 0 ? $stock / $avg7 : null;

Clasificare urgențe

  • Urgent (<7 zile stoc): days_until_stockout < 7
  • Curând (7-14 zile): days_until_stockout ∈ [7, 14)
  • Direcție trend: ↑ dacă avg7 > avg30 × 1.15; ↓ dacă avg7 < avg30 × 0.85

Parametri URL

  • coverDays: număr zile acoperire dorit (min 7, max 60, default 14)
  • selectedSupplierId: filtrare pe un furnizor specific

Acțiunea Creează Necesar

Metoda createNecesarFromSelection(array $items): primește selecția utilizatorului ([{product_id, supplier_id, qty}]), grupează pe furnizor, creează câte un PurchaseRequest per furnizor cu status submitted direct, în tranzacție DB. Redirecționează la BuyerDashboardPage.

Statistici afișate

  • Număr furnizori afectați, total produse sub stoc, produse cu stoc 0, urgente (<7 zile), curând (7-14 zile)
  • Tab "Urgente" și tab "Curând" cu lista consolidată (un rând per produs, indiferent de furnizor)
  • Secțiuni per furnizor cu produsele lor, contorizare urgente/curând/stoc 0

  2. Construiește index dublu: suppliers.email → supplier_id + supplier_contacts.email → supplier_id
  3. Listează toate folderele IMAP (recursiv), sare trash/spam/junk/drafts
  4. Per folder: reconnect fresh (evită state intern IMAP), execută SEARCH SINCE <data-3zile> via UID (evită SEARCH ALL care crăpă pe unele servere)
  5. Filtrează UID-urile deja existente în DB (EmailMessage::whereIn(imap_uid))
  6. Fetch individual per UID nou: from, subject (decode MIME), body HTML + text, to/cc/attachments (metadate)
  7. Auto-asociere furnizor: supplier.email → supplier_contact.email
  8. Salvare EmailMessage cu toate câmpurile

Output: EmailMessage records în DB. Loghează numărul de emailuri noi importate.

Algoritm:

/wp-json/malinco-erp/v1/versionGETReturnează versiunea curentă a plugin-ului instalat
/wp-json/malinco-erp/v1/product/metaGETCitește meta-datele ERP ale unui produs (supplier, sku furnizor)
/wp-json/malinco-erp/v1/product/metaPOSTScrie meta-datele ERP pe un produs
/wp-json/malinco-erp/v1/product/bulk-metaPOSTScrie meta-date pe mai multe produse simultan
/wp-json/malinco-erp/v1/cache/flushPOSTGolește cache-ul WordPress pentru produse
/wp-json/malinco-erp/v1/options/{key}GETCitește o opțiune WordPress
/wp-json/malinco-erp/v1/self-updatePOSTPrimește ZIP plugin și se auto-actualizează

Rute Chat (routes/chat.php)

RutăMetodăMiddleware / ThrottleDescriere
/chat/messagePOSTthrottle:chat-message (20 req/min per IP)Endpoint principal chatbot; input: {message, session_id}; output: {reply, session_id, products[]}
/chat/contactPOSTthrottle:chat-contact (10 req/min per IP)Salvare date contact din formularul din widget; input: {session_id, name, email, phone, wants_specialist}
/chat/configGETthrottle:chat-config (60 req/min per IP)Returnează configurarea widget-ului: primary_color, bot_name, subtitle, welcome_msg
/chat/message (OPTIONS)OPTIONS—CORS preflight
/chat/contact (OPTIONS)OPTIONS—CORS preflight
/chat/config (OPTIONS)OPTIONS—CORS preflight
