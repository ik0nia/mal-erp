Modulul 10 — Social Media & Editor Grafic

Modulul Social Media din ERP Malinco permite crearea, generarea automată prin AI și publicarea postărilor pe Facebook direct din interfața ERP. Sistemul integrează un motor de generare a imaginilor grafice bazat pe template-uri configurabile, un pipeline AI (Claude + Gemini) pentru producerea automată a conținutului și un mecanism de programare a publicării fără intervenție manuală.

Toate funcționalitățile modulului sunt accesibile din grupul de navigare Social Media din panoul App. Grupul conține: Postări, Conturi, Template-uri grafice, Editor Template și Editor Vizual.

1. Conectarea conturilor de Social Media

Pagina Conturi (Social Media → Conturi) este rezervată utilizatorilor cu rol super_admin. De aici se gestionează toate conturile Facebook conectate la ERP și se configurează cheile API necesare pentru generarea imaginilor.

1.1 Adăugarea unui cont Facebook

Apăsați butonul Adaugă cont din header-ul paginii. Va apărea un formular cu câmpurile:

  • Nume cont — eticheta afișată în ERP (ex: Malinco Facebook).
  • Facebook Page ID — ID-ul numeric al paginii de Facebook (se găsește în setările paginii sau prin Facebook Business Manager).
  • Page Access Token — token-ul de acces al paginii generat din Meta Developer Console. Câmpul este de tip parolă; valoarea se stochează criptat în baza de date.
  • Expiră la (opțional) — data de expirare a token-ului. Sistemul afișează avertisment când token-ul expiră în mai puțin de 7 zile.

Sistemul suportă atât platforma Facebook cât și Instagram. Token-ul de acces este stocat criptat (Laravel encrypt()) — nu este niciodată vizibil în text clar în baza de date.

1.2 Gestionarea token-urilor

Token-urile de acces Meta expiră periodic (de obicei la 60 de zile pentru token-uri de lungă durată). Pe fiecare card de cont din pagina Conturi există posibilitatea de a actualiza token-ul fără a recrea contul. Dacă introduceți un Meta App ID și un Meta App Secret prin butonul Setează Gemini API Key, sistemul poate genera automat token-uri permanente (non-expirabile).

Activarea/dezactivarea unui cont se face prin toggle-ul de pe cardul contului. Un cont dezactivat nu mai apare ca opțiune la crearea postărilor noi.

1.3 Configurarea cheilor API

Butonul Setează Gemini API Key deschide un formular pentru configurarea:

  • Gemini API Key — cheia Google Gemini folosită pentru generarea imaginilor grafice. Se stochează criptat în AppSetting.
  • Meta App ID și Meta App Secret — opționale, necesare pentru generarea token-urilor permanente prin OAuth.

Cheia Anthropic (Claude) folosită pentru generarea caption-urilor este configurată separat în Setări aplicație sub cheia anthropic_api_key.

1.4 Descărcarea postărilor istorice

Pentru fiecare cont există un buton Fetch posturi care pornește job-ul FetchSocialPostsJob în background. Acesta descarcă postările publicate anterior pe pagina Facebook și le salvează local ca postări fetched — acestea servesc drept material de analiză pentru profilul de stil al brandului.

2. Crearea unei postări sociale — manual

Postările sociale se gestionează din Social Media → Postări. Lista afișează toate postările cu imagine preview, tip, brief, status, data programată și data publicată.

2.1 Formularul de creare

Apăsați Postare nouă și completați formularul:

  • Cont Facebook — selectați contul pe care va fi publicată postarea. Implicit este selectat primul cont activ din sistem.
  • Tip postare — alege dintre:
    • Produs — postare centrată pe un produs din catalogul WooCommerce.
    • Brand — postare de promovare a unui brand partener (cu logo-ul brandului în grafică).
    • Promoție — postare de tip ofertă sau reducere.
    • General — postare de informare sau comunicare generală.
  • Produs (vizibil doar pentru tipul Produs) — căutare în catalogul de produse sincronizate cu WooCommerce. Câmpul este opțional; dacă nu se alege un produs, AI-ul va genera conținut conform briefului liber.
  • Brand (vizibil doar pentru tipul Brand) — selectarea brandului din baza de date.
  • Template grafic — alegerea opțională a unui template vizual predefinit. Dacă nu se alege, sistemul selectează automat template-ul potrivit tipului de postare sau template-ul implicit (malinco-default).
  • Direcție / Brief — câmp text liber obligatoriu, unde descrieți ce doriți să comunice postarea. Exemplu: Promovează cărămizile Wienerberger pentru construcții rezidențiale. Accent pe durabilitate și izolație termică.

2.2 Ce se întâmplă după salvare

Imediat după salvarea formularului, sistemul dispatch-ează automat job-ul GenerateSocialPostJob și redirectează la pagina de vizualizare a postării. O notificare informativă confirmă că generarea a pornit: "Claude și Gemini generează caption-ul și imaginea. Pagina se actualizează automat."

Pagina de vizualizare face polling automat la fiecare 3 secunde cât timp postarea are statusul Se generează... sau Se publică..., astfel că nu este necesar să reîncărcați manual pagina.

3. Generarea automată a conținutului prin AI

Întregul pipeline de generare a conținutului este automatizat și rulează în background prin sistemul de queue Laravel. Procesul are două etape principale: generarea textului (caption + hashtag-uri) și generarea imaginii grafice.

3.1 Generarea caption-ului cu Claude AI

Serviciul SocialCaptionService folosește Claude AI (Anthropic) pentru a produce:

  • Caption — textul principal al postării, adaptat stilului brandului și briefului furnizat.
  • Hashtag-uri — set de hashtag-uri relevante pentru platforma Facebook.
  • Prompt imagine — instrucțiunile pentru Gemini, descrivând imaginea ce urmează a fi generată.
  • Titlu grafică — titlul mare afișat în imaginea grafică (max. ~40 caractere).
  • Subtitlu grafică — linia secundară din grafică (max. 70 caractere).

Dacă contul are un profil de stil activ (SocialStyleProfile), acesta este injectat în promptul Claude ca context suplimentar — sistemul va genera conținut respectând tonul, vocabularul, structura caption-ului și hashtag-urile uzuale identificate anterior din postările istorice ale paginii.

3.2 Popularea automată a textelor grafice

După generarea caption-ului, job-ul rezolvă automat cele trei texte afișate în imagine:

  • Titlul graficii — dacă AI-ul a generat un titlu explicit, acesta este folosit. Altfel, sistemul preia automat numele produsului (eliminând codul de catalog numeric de la final) sau numele brandului.
  • Subtitlul graficii — dacă AI-ul nu a generat subtitlu, se folosește prima linie semnificativă din caption (cu emoji-ul de la început eliminat dacă există, și trunchiată la 70 caractere).
  • Label-ul (eyebrow) — generat automat în funcție de tipul postării: DIN CATALOGUL NOSTRU (produs), BRAND PARTENER (brand), OFERTĂ SPECIALĂ (promoție) sau MATERIALE DE CALITATE (general).

3.3 Generarea imaginii grafice cu Node.js

Imaginea grafică este produsă de serviciul NodeImageRenderer, care rulează un renderer Node.js local pe server. Renderer-ul primește configurația template-ului, textele generate și URL-ul imaginii produsului (sau calea locală a logo-ului brandului) și produce o imagine finalizată în format PNG, de dimensiune 1080×1080 pixeli (format pătrat, optim pentru Facebook/Instagram).

Imaginea produsă este salvată pe disk-ul public Laravel și calea sa este stocată în câmpul image_path al postării.

3.4 Statusurile postării

O postare parcurge următoarele stări:

  • Draft — postarea tocmai a fost creată, generarea nu a pornit încă.
  • Se generează... — job-ul de generare rulează în background.
  • Gata de programare — caption-ul și imaginea au fost generate cu succes; postarea poate fi programată sau publicată imediat.
  • Programată — postarea are o dată de publicare stabilită.
  • Se publică... — job-ul de publicare rulează.
  • Publicată — postarea a fost trimisă cu succes pe Facebook.
  • Eșuată — a apărut o eroare la generare sau publicare; detaliile erorii sunt afișate în secțiunea Eroare din pagina de vizualizare.

3.5 Acțiuni disponibile pe o postare

Din pagina de vizualizare a postării, în funcție de statusul curent, sunt disponibile:

  • Editează brief (draft/gata/eșuat) — modifică direcția postării fără a regenera.
  • Regenerează (gata/eșuat) — regenerează complet caption-ul și imaginea, cu posibilitate de a alege un alt template grafic.
  • Regenerează grafică (gata/eșuat) — regenerează doar imaginea, păstrând caption-ul existent.
  • Editează caption (gata) — editare manuală a caption-ului și hashtag-urilor generate.
  • Programează (gata) — setează data și ora publicării.
  • Reprogramează (programat) — modifică data de publicare.
  • Anulează programarea (programat) — revine la statusul Gata.
  • Publică acum (gata) — publică imediat pe Facebook, fără programare.

4. Editorul grafic de template-uri

Template-urile grafice definesc aspectul vizual al imaginilor generate. Un template controlează culorile, fonturile, elementele afișate și pozițiile lor. ERP-ul oferă două interfețe complementare pentru gestionarea template-urilor: Editorul de setări (configurare prin câmpuri de formular) și Editorul Vizual (ajustare vizuală cu drag-and-drop).

4.1 Crearea unui template nou

Navigați la Social Media → Template-uri grafice și apăsați Template nou. Completați câmpurile din secțiunea Informații generale:

  • Nume template — denumire descriptivă (ex: Produs Standard Malinco).
  • Slug — identificatorul unic intern, folosit în cod și în postări (ex: malinco-default). Trebuie să fie unic în sistem.
  • Layout — tipul de compoziție vizuală:
    • Product — imaginea produsului apare în dreapta; textele (titlu, subtitlu, label) în stânga.
    • Brand — logo-ul brandului partener apare mare în dreapta.
  • Activ — template-urile inactive nu apar ca opțiune la crearea postărilor.

5. Elementele disponibile în editor

Configurația unui template se împarte în mai multe categorii de elemente, accesibile din formularul de editare a template-ului (secțiunile colapsabile) sau din Editorul Template (pagina dedicată cu preview live).

5.1 Culori și identitate de brand

  • Culoare principală — culoarea dominantă a template-ului, folosită pentru bara de jos, fundalul label-ului și elementele de accent. Valoarea implicită este #a52a3f (roșu Malinco). Se configurează prin color picker.

5.2 Bara de jos

Bara colorată din partea inferioară a imaginii afișează informații de contact și transport:

  • Text principal — text scurt în majuscule (ex: ASIGURĂM TRANSPORT ȘI DESCĂRCARE CU MACARA). Max. 120 caractere.
  • Text secundar — adresa și datele de contact (ex: Sântandrei, Nr. 311 | www.malinco.ro | 0359 444 999). Max. 200 caractere.
  • Iconița camion (toggle) — afișează sau ascunde o pictogramă camion în bara de jos.
  • Bara curcubeu (toggle) — activează/dezactivează o bandă decorativă în culorile spectrului, plasată la baza imaginii.

5.3 Butonul CTA (Call to Action)

  • Text buton CTA — textul butonului de acțiune suprapus pe imagine (ex: malinco.ro →). Max. 60 caractere.

5.4 Logo și proporții

Secțiunea Proporții & Fonturi (colapsată implicit) permite ajustarea fină a scărilor și mărimilor de font — toate exprimate ca proporție față de lățimea imaginii (1080px = 1.0):

  • Scală logo Malinco — proporția lățimii logo-ului față de lățimea imaginii. Implicit: 0.36 (36% din lățime, adică ~389px). Interval: 0.15–0.60.
  • Mărime font titlu — proporția dimensiunii fontului titlului principal. Implicit: 0.075 (~81px la 1080px lățime). Interval: 0.04–0.12.
  • Mărime font subtitlu — proporția dimensiunii fontului subtitlului. Implicit: 0.029 (~31px). Interval: 0.015–0.06.

5.5 Imaginea produsului sau logo-ul brandului

Imaginea principală din dreapta graficii este populată automat la generare:

  • Pentru layout Product: se folosește URL-ul imaginii principale a produsului din WooCommerce (main_image_url).
  • Pentru layout Brand: se caută logo-ul brandului în directorul local storage/app/public/brand-logos/. Fișierul trebuie să fie denumit după slug-ul brandului (ex: wienerberger.png).

6. Salvarea și reutilizarea template-urilor

6.1 Salvarea setărilor

Din Editorul Template (Social Media → Editor Template), modificările sunt salvate prin butonul Salvează. Configurația completă se stochează în câmpul JSON config al modelului GraphicTemplate, incluzând: culorile, textele barei de jos, toggle-urile, scala logo-ului, proporțiile fonturilor și pozițiile elementelor.

6.2 Preview live

Butonul Generează preview din Editorul Template salvează mai întâi setările curente, apoi apelează renderer-ul Node.js pentru a produce o imagine de previzualizare cu texte demonstrative (Izolație Termică Superioară / Parteneri de încredere pentru construcții solide). Preview-ul este actualizat imediat în pagină fără reîncărcare.

Imaginea de preview este salvată și în câmpul preview_image al template-ului — aceasta apare ca thumbnail în lista de template-uri din Social Media → Template-uri grafice.

6.3 Template implicit

Sistemul caută template-ul implicit în ordinea:

  1. Template-ul cu slug-ul malinco-default.
  2. Primul template activ din baza de date (în ordine de creare).
Template-ul implicit este folosit când o postare nu are un template explicit ales și nu există niciun template activ corespunzător tipului postării.

6.4 Selecția automată a template-ului pe tip de postare

La generarea unei postări, sistemul caută automat template-ul potrivit după câmpul layout: template-urile cu layout = 'product' sunt folosite pentru postările de tip produs, iar cele cu layout = 'brand' pentru postările de brand. Dacă nu există un template specific tipului, se folosește template-ul implicit.

6.5 Acțiunea Editor Vizual

Din lista de template-uri, butonul Editor vizual deschide editorul Fabric.js într-o fereastră nouă (ruta /template-editor/{id}). Acesta permite ajustarea pozițiilor elementelor pe canvas prin drag-and-drop, direct pe imaginea randată — fără a fi necesară editarea numerică a proporțiilor.

7. Popularea automată a template-ului cu date produs (AI)

Când o postare de tip Produs este generată, sistemul populează automat grafică cu datele din WooCommerce, fără intervenție manuală:

7.1 Titlul produsului

Numele produsului este procesat înainte de a fi afișat în grafică: codul numeric de catalog de la finalul numelui este eliminat automat prin regex (ex: Cărămidă Porotherm 30 P+W 123/456 devine Cărămidă Porotherm 30 P+W). Aceasta previne afișarea codurilor tehnice irelevante publicului.

7.2 Imaginea produsului

URL-ul imaginii principale a produsului (main_image_url) este transmis direct renderer-ului Node.js, care o descarcă și o compozitează în grafică. Dacă produsul nu are imagine, slot-ul rămâne gol (renderer-ul gestionează lipsa imaginii fără eroare).

7.3 Subtitlul generat din caption

Dacă AI-ul (Claude) nu a generat explicit un subtitlu pentru grafică, sistemul extrage prima linie semnificativă din caption generat și o folosește ca subtitlu. Procesul include:

  • Eliminarea emoji-urilor de la începutul liniei.
  • Trunchiere la 70 de caractere dacă linia este prea lungă.
  • Folosirea primei linii non-vide din caption.

7.4 Logo brand în postările de tip Brand

Pentru postările de brand, sistemul caută automat logo-ul în două locuri:

  1. Dacă brandul are un logo_url setat, se extrage numele fișierului din URL și se caută fișierul local în storage/app/public/brand-logos/.
  2. Dacă produsul are un câmp brand completat, se construiește un slug din numele brandului (minuscule, fără caractere speciale) și se caută fișierul brand-logos/{slug}.png.

8. Programarea postărilor

Modulul include un sistem complet de programare a publicării, cu protecție împotriva dublei publicări și suport pentru reprogramare.

8.1 Programarea unei postări

Din pagina de vizualizare a unei postări cu statusul Gata de programare, apăsați butonul Programează. Va apărea un dialog cu un câmp date-time pentru selectarea momentului publicării. Data minimă acceptată este cu 5 minute în viitor. Valoarea implicită propusă este ora rotundă imediat următoare (now + 1 oră, rotunjit la oră întreagă).

La confirmare, postarea trece în statusul Programată și data este salvată în câmpul scheduled_at.

8.2 Publicarea automată la momentul programat

Job-ul SocialPublishScheduledCommand (rulat din schedule-ul Laravel) verifică periodic postările cu statusul Programată și scheduled_at ≤ now() și dispatch-ează pentru fiecare PublishSocialPostJob.

Job-ul de publicare folosește Meta Graph API prin clientul MetaGraphClient pentru a posta imaginea și caption-ul complet (caption + hashtag-uri separate prin linie goală) pe pagina Facebook. Procesul este învelit în tranzacție cu lockForUpdate() pentru a preveni dubla publicare în caz de retry.

La publicare cu succes, se salvează:

  • platform_post_id — ID-ul postării pe Facebook.
  • platform_url — URL-ul direct al postării pe Facebook (https://www.facebook.com/{page_id}/posts/{post_id}).
  • published_at — timestamp-ul publicării.

Job-ul are 3 încercări cu un interval de 60 de secunde între ele. Dacă toate eșuează, postarea trece în statusul Eșuată cu mesajul de eroare salvat.

8.3 Publicarea imediată

Dacă nu doriți să programați, butonul Publică acum setează scheduled_at = now() și dispatch-ează imediat PublishSocialPostJob. Postarea trece din statusul Gata în Se publică... și în câteva secunde devine Publicată (pagina face polling automat la 3s).

8.4 Reprogramarea și anularea

  • Reprogramează — disponibil pe postările cu statusul Programată; modifică scheduled_at fără a schimba statusul.
  • Anulează programarea — revine postarea la statusul Gata de programare și șterge scheduled_at. Necesită confirmare.

9. Gestionarea stilului vizual al brandului

Sistemul poate analiza automat stilul vizual și editorial al paginilor de Facebook conectate și poate construi un profil de stil (SocialStyleProfile) care ghidează generarea viitoare a conținutului.

9.1 Descărcarea postărilor istorice

Primul pas este descărcarea postărilor publicate anterior pe pagina Facebook prin butonul Fetch posturi de pe cardul contului (pagina Conturi). Job-ul FetchSocialPostsJob accesează Meta Graph API și salvează postările istorice local ca modele SocialFetchedPost.

9.2 Analiza stilului editorial

Butonul Analizează stil de pe cardul contului dispatch-ează AnalyzeSocialStyleJob în background. Acest job trimite postările istorice la Claude AI pentru analiză și extrage:

  • Ton — tonul editorial predominant (profesional, prietenos, entuziast, tehnic etc.).
  • Vocabular și stil — tipul de cuvinte folosite, nivelul de formalitate, structuri sintactice preferate.
  • Hashtag-uri uzuale — pattern-urile de hashtag-uri din postările anterioare.
  • Structura caption-ului — cum sunt organizate textele (intro + corp + CTA, liste, întrebări etc.).
  • Stil vizual — observații despre compoziție, culori și stilul grafic al postărilor.

Profilul rezultat este salvat ca SocialStyleProfile activ pentru cont și este injectat automat în promptul Claude la generarea oricărei postări noi pentru acel cont, asigurând coerența comunicării în timp.

9.3 Referințe vizuale și generarea template-urilor din imagini

Pagina Conturi permite încărcarea de imagini de referință vizuală prin butonul Încarcă referințe vizuale. Imaginile (PNG/JPG/WEBP, max. 5MB fiecare, max. 20 fișiere) sunt stocate în storage/app/public/social/style-references/.

Aceste imagini reprezintă materiale grafice cu stilul vizual dorit pentru brand — pot fi postări anterioare de succes, materiale de la agenții partenere sau orice referință grafică relevantă.

Selectând una sau mai multe imagini de referință și apăsând Generează template-uri, se dispatch-ează AnalyzeReferenceImagesJob cu imaginile selectate. Job-ul trimite imaginile la Claude AI pentru analiză vizuală și generează automat configurații de template-uri inspirate din stilul detectat. Utilizatorul este notificat prin Filament Notification la finalul analizei.

9.4 Profil de stil activ

Un cont poate avea mai multe profile de stil generate de-a lungul timpului. La orice moment, doar unul este marcat ca is_active = true și este utilizat la generarea postărilor. Profilul activ este cel mai recent generat. Câmpul style_analyzed_at din contul social înregistrează momentul ultimei analize, util pentru a ști dacă profilul este actualizat.

9.5 Editorul Vizual (Fabric.js)

Pe lângă editorul de setări bazat pe formulare, sistemul include un Editor Vizual cu canvas interactiv Fabric.js, accesibil din butonul Editor vizual al unui template sau din meniul Social Media → Editor Vizual.

Editorul vizual afișează imaginea de preview a template-ului randată și permite ajustarea pozițiilor elementelor prin drag-and-drop direct pe canvas. Pozițiile ajustate sunt salvate în câmpul config.element_positions al template-ului (structură JSON).

La apăsarea butonului de salvare, editorul:

  1. Trimite pozițiile curente ale elementelor (JSON) la server prin Livewire.
  2. Serverul actualizează config.element_positions în baza de date.
  3. Se apelează imediat renderer-ul Node.js pentru a genera un preview actualizat cu noile poziții.
  4. Preview-ul nou este afișat în editor fără a reîncărca pagina.

Editorul vizual suportă comutarea între template-uri prin dropdown fără a reîncărca pagina — evenimentul Livewire template-loaded actualizează canvas-ul Alpine.js cu pozițiile și preview-ul noului template.

9.6 Arhitectura editorului Fabric.js (detalii tehnice)

Editorul este implementat ca o aplicație JavaScript modulară, structurată în mai multe module specializate:

  • CanvasManager + ZoomManager — gestionează canvas-ul Fabric.js și nivelul de zoom (stabilizate, stabile în producție).
  • StateManager + ElementFactory — gestionează starea editorului și crearea elementelor noi (text, imagini, forme geometrice, background).
  • EditorCore + Toolbar — logica centrală a editorului și toolbar-ul cu acțiuni contextuale bazate pe tipul elementului selectat.
  • LayersPanel — panoul de straturi (layers) cu drag-and-drop pentru reordonarea elementelor, complet driven de state.

10. Fluxul complet — de la brief la publicare

Mai jos este rezumatul fluxului end-to-end pentru o postare tipică de produs:

  1. Configurare inițială (o singură dată per brand): conectare cont Facebook, setare Gemini API Key, încărcare referințe vizuale, analiză stil, creare/configurare template.
  2. Creare postare: Social Media → Postări → Postare nouă → alege cont, tip Produs, selectează produsul, completează brief-ul → Salvează.
  3. Generare automată: sistemul dispatch-ează GenerateSocialPostJob; Claude generează caption + texte grafice; Node.js renderer produce imaginea 1080×1080; postarea trece în statusul Gata de programare.
  4. Revizuire: pagina se actualizează automat (polling 3s); revizuiți imaginea și caption-ul generat; dacă este necesar, editați caption-ul sau regenerați integral.
  5. Programare sau publicare imediată: apăsați Programează și alegeți momentul optim sau Publică acum pentru publicare imediată.
  6. Publicare: PublishSocialPostJob trimite postarea pe Facebook prin Meta Graph API; se salvează URL-ul și ID-ul postării publicate.

11. Filtrare și căutare în lista de postări

Lista de postări (Social Media → Postări) include filtre rapide pentru:

  • Status — filtrare după oricare din cele 7 stări ale postării.
  • Tip postare — filtrare după Produs, Brand, Promoție sau General.

Sortarea implicită este descrescătoare după data creării (cele mai noi primele). Coloana Creat la este disponibilă ca coloană toggleabilă ascunsă implicit. Coloanele Programată și Publicată sunt sortabile.

Preview-ul imaginii este afișat direct în tabel ca thumbnail pătrat de 60×60 pixeli, permițând o scanare vizuală rapidă a conținutului.
