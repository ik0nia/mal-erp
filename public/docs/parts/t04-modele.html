Model: WooProduct

Tabelă: woo_products

Relații:

  • connection() — belongsTo(IntegrationConnection) — conexiunea WooCommerce/WinMentor din care provine produsul
  • attributes() — hasMany(WooProductAttribute) — atributele produsului (ordonate după position)
  • categories() — belongsToMany(WooCategory) prin pivot woo_product_category — categoriile WooCommerce
  • stocks() — hasMany(ProductStock) — stocuri per locație/depozit
  • priceLogs() — hasMany(ProductPriceLog) — istoricul modificărilor de preț
  • dailyStockMetrics() — hasMany(DailyStockMetric, FK: sku) — metrici zilnice stoc pentru grafice
  • latestDailyStockMetric() — hasOne(DailyStockMetric) ofMany(max day) — cel mai recent metric zilnic
  • suppliers() — belongsToMany(Supplier) prin pivot product_suppliers using ProductSupplier — furnizori cu prețuri și condiții
  • offerItems() — hasMany(OfferItem) — rânduri de oferte care referențiază acest produs

Câmpuri fillable:

connection_id, woo_id, type, status, sku, name, winmentor_name, slug, short_description, description, regular_price, sale_price, price, stock_status, manage_stock, woo_parent_id, main_image_url, data (JSON), source, is_placeholder, unit, brand, weight, dim_length, dim_width, dim_height, erp_notes, product_type, procurement_type, on_demand_label, is_discontinued, discontinued_reason

Casts:

  • connection_id, woo_id, woo_parent_id — integer
  • manage_stock, is_placeholder, is_discontinued — boolean
  • data — array (JSON WooCommerce raw)

Constante/Stări:

  • SOURCE_WOOCOMMERCE / SOURCE_WINMENTOR_CSV — sursa de import
  • TYPE_SHOP / TYPE_PRODUCTION / TYPE_PALLET_FEE — tipul produsului
  • PROCUREMENT_STOCK / PROCUREMENT_ON_DEMAND — tipul de aprovizionare (stoc vs. la comandă)

Metode business importante:

  • productTypeOptions() — returnează array label-uri pentru tipul produsului
  • productTypeBadgeColor(string $type) — returnează culoarea badge Filament în funcție de tip
  • getDecodedNameAttribute() — accessor: decodează entitățile HTML din nume (ex: &amp; → &)

Model: PurchaseRequest

Tabelă: purchase_requests

Traits: HasLocationScope, HasStatusEnum

Relații:

  • user() — belongsTo(User) — utilizatorul care a creat necesarul
  • location() — belongsTo(Location) — locația pentru care s-a întocmit necesarul
  • items() — hasMany(PurchaseRequestItem) — liniile necesarului
  • wooOrder() — belongsTo(WooOrder) — comanda WooCommerce sursă (dacă source_type = woo_order)

Câmpuri fillable:

number, user_id, location_id, status, notes, woo_order_id, source_type

Casts:

  • user_id, location_id — integer

Constante/Stări:

  • Status: STATUS_DRAFT → STATUS_SUBMITTED → STATUS_PARTIALLY_ORDERED / STATUS_FULLY_ORDERED / STATUS_CANCELLED
  • Sursă: SOURCE_MANUAL / SOURCE_WOO_ORDER

Metode business importante:

  • getOrCreateDraft(User $user) — firstOrCreate: obține sau creează unicul draft activ al unui utilizator
  • createFromWooOrder(WooOrder $order) — creează automat un PNR submitted din produsele on_demand dintr-o comandă WooCommerce; idempotent
  • recalculateStatus() — recalculează statusul cererii în funcție de starea liniilor (ordered_quantity vs quantity): FULLY_ORDERED / PARTIALLY_ORDERED / SUBMITTED / CANCELLED
  • generateNumber() — (privat) generează număr de serie PNR-XXXX pe baza seriei din AppSetting
  • statusLabels() / statusColorMap() — mapări statice pentru UI Filament

Hooks booted:

creating: auto-generare număr, setare status draft, auto-completare user_id și location_id din sesiunea curentă.

Model: PurchaseOrder

Tabelă: purchase_orders

Traits: HasStatusEnum

Relații:

  • supplier() — belongsTo(Supplier)
  • buyer() — belongsTo(User, buyer_id) — achizitorul care a creat PO-ul
  • approvedBy() — belongsTo(User, approved_by)
  • rejectedBy() — belongsTo(User, rejected_by)
  • receivedBy() — belongsTo(User, received_by)
  • items() — hasMany(PurchaseOrderItem) — liniile comenzii de achiziție

Câmpuri fillable:

number, supplier_id, buyer_id, status, total_value, currency, notes_internal, notes_supplier, approved_by, approved_at, rejected_by, rejected_at, rejection_reason, sent_at, received_at, received_by, received_notes

  • approved_by, rejected_by, received_by — integer

Constante/Stări:

  • STATUS_DRAFT → STATUS_PENDING_APPROVAL → STATUS_APPROVED / STATUS_REJECTED → STATUS_SENT → STATUS_RECEIVED / STATUS_CANCELLED

Metode business importante:

  • recalculateTotals() — recalculează total_value ca sumă a line_total din items (saveQuietly)
  • needsApproval() — verifică dacă PO necesită aprobare: (1) total depășește po_approval_threshold al furnizorului SAU (2) oricare item depășește po_max_qty din ProductSupplier; evită N+1 prin preîncărcarea ProductSupplier în batch
  • generateNumber() — (privat) generează număr PO-Ymd-XXXX bazat pe MAX sequence a zilei curente
  • statusLabels() / statusColorMap() — mapări statice pentru UI Filament

Hooks booted:

creating: auto-generare număr, status implicit draft, monedă implicită RON, auto-completare buyer_id din sesiunea curentă.
