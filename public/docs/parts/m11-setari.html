Modul 11 — Setări și Administrare

Secțiunea de setări și administrare centralizează toate configurările tehnice și operaționale ale platformei ERP Malinco. Accesul la aceste pagini este rezervat utilizatorilor cu rol super_admin sau admin. Modificările produc efect imediat, fără a fi necesară repornirea aplicației.

1. Pagina Setări Aplicație

Pagina Setări aplicație se găsește în meniul lateral al panoului App, identificată prin iconița roată dințată (cog-6-tooth). Este organizată în patru secțiuni distincte: Branding, Email inbox (IMAP), WooCommerce Plugin Bridge și Inteligență Artificială (Claude).

Toate câmpurile de tip parolă sau cheie API sunt stocate criptat în baza de date (prin mecanismul Laravel encrypt()). Din acest motiv, la deschiderea paginii câmpurile de parolă apar goale — sistemul nu reîncarcă valorile sensibile în formular. Pentru a modifica o parolă sau cheie, introduceți noua valoare; dacă lăsați câmpul gol, valoarea existentă se păstrează nemodificată.

1.1 Secțiunea Branding

CâmpDescriereObservații
Nume aplicație Textul afișat în bara de navigare și în titlul paginilor ERP. Valoare implicită: Malinco ERP. Maxim 100 caractere.
Logo Imaginea afișată în header-ul panoului de administrare. Format recomandat: PNG cu fundal transparent, înălțime minimă 200px. Dimensiunea maximă acceptată este 2 MB. Fișierul se salvează în directorul storage/app/public/brand/.

2. Configurarea IMAP — Email Inbox

Modulul de inteligență email necesită o conexiune IMAP pentru a citi mesajele din căsuța poștală a companiei. Conexiunea este exclusiv read-only: sistemul nu trimite, nu șterge și nu mută niciun email. Inbox-ul din Outlook sau webmail rămâne complet neafectat.

Emailurile sunt preluate automat la fiecare 5 minute printr-un job programat (FetchEmailsJob). Sistemul folosește comanda IMAP SINCE <data> pentru a prelua doar mesajele noi, evitând descărcarea repetată a întregului inbox.

Câmpuri de configurare IMAP

CâmpDescriereValoare tipică
Server IMAP Hostname-ul serverului de mail al companiei. mail.malinco.ro
Port Portul TCP pentru conexiunea IMAP. 993 (SSL standard)
Criptare Protocolul de securitate al conexiunii. Opțiuni disponibile:
  • SSL — portul 993, criptare imediată (recomandat)
  • TLS — portul 143, criptare prin negociere
  • STARTTLS — upgrade la TLS pe conexiune necriptată
  • Fără criptare — numai pentru testare în rețea internă

După salvarea configurației IMAP, puteți testa conexiunea executând comanda: 

php artisan email:import-history --folder=INBOX --limit=10

3. Configurarea API Anthropic (Claude AI)

Platforma ERP integrează modelele de limbaj Claude de la Anthropic pentru mai multe funcționalități automate: procesarea emailurilor de la furnizori, chatbot-ul de pe site, rezumatele conversațiilor, analiza BI, îmbogățirea produselor și generarea de conținut pentru Social Media.

Obținerea cheii API

  1. Accesați console.anthropic.com
  2. Autentificați-vă cu contul organizației
  3. Navigați la API Keys și creați o cheie nouă
  4. Copiați cheia (format: sk-ant-api03-...) — aceasta nu mai poate fi vizualizată ulterior

Configurare în ERP

Introduceți cheia în câmpul Anthropic API Key din secțiunea Inteligență Artificială (Claude) și salvați. Cheia este stocată criptat. Dacă nu introduceți nicio valoare, sistemul folosește automat variabila de mediu ANTHROPIC_API_KEY din fișierul .env (fallback).

Modele utilizate și costuri estimative

ModelPreț input (per M tokens)Preț output (per M tokens)Utilizare principală
claude-haiku-4-5 $0.80 $4.00 Chatbot site, procesare emailuri, rezumate chat
claude-sonnet-4-6 $3.00 $15.00 Analiză BI, rapoarte complexe
claude-opus-4-6 $15.00 $75.00 Sarcini analitice de înaltă complexitate

4. Configurarea Integrărilor WooCommerce

Integrările cu platforme externe se gestionează din panoul Admin, secțiunea Integration Connections. Fiecare integrare este asociată unei locații (depozit/magazin) și poate fi de unul din trei tipuri: WooCommerce, WinMentor CSV sau Sameday Courier.

4.1 Câmpuri comune tuturor integrărilor

CâmpDescriere
Locație Depozitul sau magazinul căruia îi aparține această integrare.
Tip (Provider) WooCommerce, WinMentor CSV sau Sameday Courier.
Nume Etichetă descriptivă pentru identificarea rapidă a conexiunii.
Activ Activează sau dezactivează sincronizarea automată pentru această conexiune.

4.2 Integrare WooCommerce

Conexiunea WooCommerce necesită credențiale REST API generate din panoul WordPress (WooCommerce → Settings → Advanced → REST API).

CâmpDescriere
URL de bază Adresa site-ului WooCommerce, ex: https://malinco.ro. Slash-ul final este eliminat automat.
Consumer Key Cheia publică REST API WooCommerce (format ck_...). Stocată criptat.
Consumer Secret Cheia secretă REST API WooCommerce (format cs_...). Stocată criptat.
Verificare SSL Activați în producție. Dezactivați doar pentru testare locală cu certificate self-signed.
Webhook Secret Cheia secretă pentru validarea webhook-urilor primite de la WooCommerce. Se generează automat la crearea conexiunii. URL-ul webhook este https://erp.malinco.ro/webhooks/woo/{id_conexiune}. Stocată criptat.

4.3 Integrare WinMentor CSV

Importul din WinMentor se realizează prin fișiere CSV exportate periodic. Câmpurile Consumer Key și Consumer Secret nu sunt utilizate pentru această integrare — doar URL-ul fișierului CSV este necesar, configurat în câmpul settings → csv_url.

Opțiuni suplimentare în setările JSON ale conexiunii:

  • auto_sync_enabled: activează importul automat periodic (boolean)
  • sync_interval_minutes: intervalul de sincronizare, minim 5 minute (integer)

4.4 Integrare Sameday Courier

Conexiunea Sameday folosește câmpurile Consumer Key drept username și Consumer Secret drept parolă pentru API-ul Sameday. URL-ul de bază poate fi lăsat gol (sistemul folosește automat https://api.sameday.ro pentru producție sau https://sameday-api.demo.zitec.com pentru mediul demo, configurat prin settings → environment).

5. Setarea Cheii API pentru Plugin-ul WordPress

Plugin-ul malinco-erp-bridge instalat pe site-ul WordPress comunică cu ERP-ul printr-un API REST securizat. Autentificarea se face prin header-ul HTTP X-ERP-Api-Key, iar cheia trebuie să fie identică în ambele sisteme.

Pași de configurare

  1. În ERP: introduceți cheia secretă în câmpul API Key plugin din secțiunea WooCommerce Plugin Bridge a paginii Setări aplicație și salvați. Cheia este stocată criptat în tabela app_settings sub cheia woo_plugin_api_key.
  2. Pe WordPress: accesați WP Admin → Settings → ERP Bridge și introduceți exact aceeași cheie în câmpul corespunzător.

Cheia poate fi orice șir alfanumeric aleatoriu. Recomandăm generarea unui șir de minimum 32 caractere aleatorii. Dacă cele două chei nu coincid, toate apelurile API vor eșua cu eroare de autentificare.

După configurare, verificați conexiunea cu comanda: 

php artisan woo:deploy-plugin --check

Rezultatul afișat va confirma versiunea plugin-ului activ și URL-ul site-ului.

6. Gestionarea Permisiunilor pe Roluri

Sistemul de control al accesului folosește un model de permisiuni bazat pe roluri (Role-Based Access Control). Permisiunile sunt stocate în tabela role_permissions și sunt incluse în cache cu un TTL de 5 minute pentru performanță optimă.

6.1 Structura permisiunilor

Fiecare înregistrare de permisiune definește ce poate face un rol asupra unui anumit resource (resursă Filament sau pagină). Câmpurile disponibile sunt:

CâmpDescriere
role Rolul utilizatorului căruia i se aplică regula (ex: buyer, sales, manager).
resource Clasa PHP completă a resursei Filament sau identificatorul paginii (ex: App\Filament\App\Resources\PurchaseOrderResource).
can_access Poate accesa (vedea în meniu) resursa respectivă.
can_create Poate crea înregistrări noi în acea resursă.
can_edit Poate edita înregistrările existente.
can_delete Poate șterge înregistrările.
can_view Poate deschide pagina de detaliu (View) a unei înregistrări.

6.2 Ierarhia rolurilor

Sistemul aplică o politică default-deny: dacă nu există nicio regulă definită explicit pentru un rol și o resursă, accesul este refuzat automat. Excepțiile sunt:

  • super_admin — acces complet la toate resursele și paginile, fără a fi necesare reguli explicite. Poate configura setările aplicației, gestionează permisiunile altor roluri și are acces la logurile de utilizare AI.
  • admin — acces complet la toate resursele, similar cu super_admin, cu excepția setărilor de sistem de nivel înalt.
  • Alte roluri (buyer, sales, manager etc.) — accesul este determinat exclusiv de regulile definite în tabela role_permissions.

6.3 Modificarea permisiunilor

Permisiunile se gestionează direct din panoul Admin al aplicației sau prin intervenție directă în baza de date (pentru configurări inițiale). La salvarea unei permisiuni, cache-ul rolului respectiv este invalidat automat, astfel că modificările devin active imediat pentru noile sesiuni.

6.4 Ordinea navigației

Pe lângă permisiuni, super_admin poate configura ordinea grupurilor și elementelor din meniul lateral. Aceste setări sunt stocate tot în app_settings, sub chei de forma nav_group_sort_* și nav_item_sort_*. Grupurile configurabile sunt:

  • Administrare magazin (sort implicit: 1)
  • Achiziții (sort implicit: 2)
  • Vânzări (sort implicit: 3)
  • Comenzi (sort implicit: 4)
  • Livrare (sort implicit: 5)
  • Rapoarte (sort implicit: 6)
  • Produse (sort implicit: 7)
  • Comunicare (sort implicit: 8)
  • Social Media (sort implicit: 9)

7. Log Utilizare AI — Monitorizare Costuri

Pagina Consum API Claude (grupul Sistem din panoul Admin) oferă vizibilitate completă asupra apelurilor API către Anthropic și costurile asociate. Aceasta este utilă pentru a identifica sursele de consum ridicat și pentru a estima bugetele lunare de utilizare AI.

7.1 Statistici afișate

IndicatorDescriere
Cost total Suma costurilor în USD pentru toate apelurile din perioada selectată.
Total tokeni input Numărul de tokeni trimiși către model (prompturi, context, date).
Total tokeni output Numărul de tokeni generați de model (răspunsuri, analize).
Număr apeluri Câte apeluri API au fost efectuate în perioada selectată.
Cost mediu pe zi Media zilnică a costurilor, calculată automat în funcție de perioada aleasă.

7.2 Filtrare pe perioade

Sunt disponibile filtrele: Azi, Ieri, Ultimele 7 zile, Ultimele 14 zile și Ultimele 30 zile.

7.3 Defalcare pe surse

Consumul este defalcat pe fiecare sursă de utilizare, oferind transparență deplină. Sursele monitorizate includ:

  • Chatbot site — răspunsuri generate pentru vizitatorii site-ului
  • Email AI — procesarea automată a emailurilor de la furnizori
  • Rezumat chat — generarea rezumatelor conversațiilor din chat
  • Domain matching furnizori — asocierea automată a domeniilor email cu furnizorii
  • BI — Rapoarte (zilnic, săptămânal, lunar, trimestrial, semestrial, anual)
  • Enrichment produse — descrieri, titluri, atribute, categorii, imagini, dimensiuni, web scraping
  • Social Media — generare caption-uri, analiză stil, generare imagini, template-uri grafice

7.4 Evoluție pe zile

A doua tabelă din pagină afișează consumul agregat pe fiecare zi din perioada selectată (cost, tokeni, număr apeluri), permițând identificarea zilelor cu activitate neobișnuit de ridicată.

7.5 Log detaliat

Ultimele 200 de înregistrări din perioada selectată sunt afișate în ordine cronologică inversă, cu detalii despre sursă, model utilizat, număr de tokeni și costul individual al fiecărui apel.

Costurile sunt calculate automat de sistem pe baza prețurilor publice Anthropic: tokenii de intrare și ieșire se contorizează separat, iar costul fiecărui apel se calculează cu formula: cost = (input_tokens / 1.000.000 × preț_input) + (output_tokens / 1.000.000 × preț_output).

8. Branduri — Gestionarea Listei de Branduri

Modulul Branduri se găsește în grupul Achiziții din meniul lateral (sort implicit: 5) și permite gestionarea catalogului de mărci comercializate de companie. Brandurile sunt utilizate în modulul de achiziții pentru a asocia furnizorii cu mărcile pe care le distribuie.

8.1 Câmpurile unui brand

CâmpDescriereObservații
Nume brand Denumirea comercială a brandului (ex: Samsung, Bosch, Philips). Obligatoriu. Maxim 255 caractere.
Slug Identificatorul URL-friendly al brandului, generat automat din nume (ex: samsung, bosch). Obligatoriu. Trebuie să fie unic în sistem. Generat automat la completarea numelui.

8.2 Asocierea furnizorilor

Fiecare brand poate fi asociat cu unul sau mai mulți furnizori care distribuie produsele acelui brand. Asocierea se face printr-o relație many-to-many. Din lista de branduri se pot vedea direct furnizorii asociați fiecărui brand (maxim 3 afișați, restul indicat prin număr).

8.3 Operațiuni disponibile

  • Creare: butonul Adaugă brand din colțul dreapta-sus al listei.
  • Editare: click pe iconița de editare din coloana de acțiuni.
  • Ștergere în masă: selectați mai multe branduri din listă și folosiți acțiunea Delete selected.
  • Filtrare după status: filtrul Activ permite afișarea doar a brandurilor active, inactive sau ambele.
  • Sortare implicită: alfabetic după numele brandului.

9. Plugin WordPress — malinco-erp-bridge

9.1 Ce face plugin-ul

Plugin-ul malinco-erp-bridge (versiunea curentă: 2.1.0) extinde WordPress-ul cu un API REST privat care permite comunicarea bidirecțională între ERP și site-ul WooCommerce. Plugin-ul acționează ca un pod (bridge) prin care ERP-ul poate citi și scrie date direct în WordPress fără a folosi API-ul standard WooCommerce REST pentru operațiunile pe meta-date.

Funcționalitățile principale ale plugin-ului includ:

  • Sincronizare meta-date produse: ERP-ul poate scrie meta-date personalizate pe produsele WooCommerce (furnizor, SKU furnizor, stoc WinMentor, prețuri, disponibilitate).
  • Sincronizare în bulk: endpoint-ul /product/bulk-meta permite actualizarea meta-datelor pentru sute de produse simultan, reducând numărul de apeluri API.
  • Logică de disponibilitate și preț: plugin-ul calculează automat starea de disponibilitate a produselor pe baza stocului WinMentor, prețului furnizorului și timpilor de livrare configurați per furnizor.
  • Golire cache: endpoint-ul /cache/flush invalidează cache-ul WordPress după modificări majore de date.
  • Citire opțiuni WordPress: ERP-ul poate citi opțiuni din baza de date WordPress prin endpoint-ul /options/{cheie}.
  • Auto-actualizare: plugin-ul se poate actualiza singur prin endpoint-ul /self-update, fără intervenție manuală în panoul WordPress.

9.2 Endpoint-uri REST disponibile

Toate endpoint-urile sunt disponibile la prefixul /wp-json/malinco-erp/v1/ și necesită autentificare prin header-ul X-ERP-Api-Key.

MetodăEndpointFuncție
GET /version Verifică că plugin-ul răspunde și returnează versiunea și URL-ul site-ului.
GET /product/meta Citește meta-datele unui produs specificat prin SKU sau ID WooCommerce.
POST /product/meta Actualizează meta-datele unui singur produs.
POST /product/bulk-meta Actualizează meta-datele pentru mai multe produse simultan.
POST /product/recalculate Declanșează recalcularea disponibilității și prețului pentru produse specificate.
POST /cache/flush Golește cache-urile WordPress (object cache, transients).
GET /options/{cheie} Citește o opțiune din baza de date WordPress.
POST /self-update Primește un fișier ZIP și actualizează plugin-ul la noua versiune.

9.3 Instalarea inițială

  1. Descărcați fișierul malinco-erp-bridge.zip din directorul public/ al ERP-ului sau construiți-l cu comanda: 
    php artisan woo:deploy-plugin --build-only
  2. În panoul WordPress, accesați Plugins → Add New → Upload Plugin, selectați ZIP-ul și instalați.
  3. Activați plugin-ul.
  4. Accesați Settings → ERP Bridge și introduceți cheia API (aceeași configurată în ERP la pasul descris în secțiunea 5).

9.4 Actualizarea plugin-ului

Actualizările pot fi aplicate automat prin comanda de deploy din ERP, fără a accesa panoul WordPress: 

php artisan woo:deploy-plugin

Comanda construiește un ZIP cu ultima versiune a fișierelor din directorul wordpress-plugin/malinco-erp-bridge/, îl trimite prin endpoint-ul /self-update și verifică versiunea activă după instalare.

Opțiuni disponibile pentru comanda de deploy:

OpțiuneEfect
--check Verifică versiunea curentă a plugin-ului activ pe WordPress, fără a face nicio modificare.
--build-only Construiește fișierul ZIP local (în storage/app/) fără a-l trimite pe WordPress. Util pentru inspecție manuală sau instalare prin interfața WordPress.

9.5 Meta-datele sincronizate

Principalele meta-date gestionate de plugin pe produsele WooCommerce:

  • _wm_stock — stocul curent din WinMentor (număr întreg)
  • _wm_price — prețul curent din WinMentor (număr zecimal)
  • _supplier_price — prețul din feed-ul furnizorului
  • _furnizor_nume — numele furnizorului principal al produsului
  • _furnizor_sku — codul SKU al produsului în catalogul furnizorului
  • _ever_in_stock — dacă produsul a fost vreodată în stoc (0 sau 1)
  • _pricing_source — sursa activă a prețului (winmentor sau supplier)
  • _stock_label — eticheta de disponibilitate afișată pe site
  • _availability_state — starea internă de disponibilitate (pentru debugging)
  • _lead_time_override_days și _lead_time_override_text — timp de livrare personalizat per produs

9.6 Sincronizarea meta furnizor → WooCommerce

La salvarea unui produs din interfața ERP (pagina de editare produs), dacă produsul are un ID WooCommerce asociat, se declanșează automat job-ul SyncProductSupplierMetaJob. Acesta scrie meta-datele _furnizor_nume și _furnizor_sku prin API-ul WooCommerce standard (WooClient::updateProduct()). Pentru sincronizarea în masă a tuturor produselor, folosiți comanda: 

php artisan woo:sync-supplier-meta

Rezumat — Fluxul de configurare recomandat

La instalarea inițială a platformei ERP sau la configurarea unui mediu nou, ordinea recomandată de configurare este:

  1. Setați Branding (nume aplicație și logo).
  2. Configurați IMAP pentru importul emailurilor istorice și sincronizarea continuă.
  3. Introduceți cheia API Anthropic pentru a activa procesarea AI a emailurilor și celelalte funcționalități inteligente.
  4. Creați integrările WooCommerce și WinMentor CSV în panoul Admin.
  5. Instalați și configurați plugin-ul WordPress cu cheia API corespunzătoare.
  6. Definiți permisiunile pe roluri pentru fiecare tip de utilizator din organizație.
  7. Monitorizați periodic consumul API Claude pentru optimizarea costurilor.
